2026-03-28：修复 packaged 桌面首页客户端异常。现象：Electron 窗口显示 `Application error: a client-side exception has occurred while loading 127.0.0.1`，Playwright 控制台报 `ChunkLoadError`，`/_next/static/chunks/app/%5Blocale%5D/{layout,page}-*.js` 先 307 后 404。根因：desktop/runtime/static-asset-dispatcher.cjs 的 resolveSafeFile() 直接拿 URL pathname 片段做文件查找，没有对 `%5Blocale%5D` 做 decodeURIComponent，导致磁盘存在的 `.next/static/chunks/app/[locale]/*.js` 被误判为不存在，随后 engine-runtime-server 的 locale redirect 把请求重写到 `/zh/_next/...`，最终 404。修复：在 resolveSafeFile() 中先 decodeURIComponent(relativePath) 再做安全归一化与根目录逃逸校验。回归保护：scripts/desktop/smoke-test.mjs 新增 verifyCriticalClientChunks()，从 /zh HTML 中抽取 `/_next/static/chunks/app/%5Blocale%5D/*.js` 并逐个断言 200 + javascript content-type，避免再次出现“页面 HTML 正常但 hydration chunk 404”的假阳性。验证：修复后 Playwright 打开 http://127.0.0.1:13000/zh 页面标题恢复为 `AI动漫制作工作台`，控制台错误清零；npm run desktop:smoke:packaged 通过。
